    React Hook 基础

    槐序 2021/11/12 reactreact-hook

    # 介绍

    # 定义

    Hook 是一些可以让你在函数组件里“钩入” React state 及生命周期等特性的函数。

    # 优点

    • 比 render props (opens new window) 和 高阶组件 (opens new window)更优秀的逻辑复用方案（自定义hook）
    • Hook 将组件中相互关联的部分拆分成更小的函数（useEffect）
    • 易于理解，Hook 使你在非 class 的情况下可以使用更多的 React 特性

    # hook规则

    必须在自定义hook或函数组件的最顶层使用hook，不要在循环、条件或嵌套函数中调用hook。因为react依赖hook的调用顺序来关联内部state和对应hook。

    # useState

    # 介绍

    接受一个初始值，返回当前状态和一个让你更新它的函数。

     import React, { useState } from 'react';
 
 function Example() {
   // 声明一个叫 "count" 的 state 变量
   const [count, setCount] = useState(0);
   return (
     <div>
       <p>You clicked {count} times</p>
       // setCount传入的参数和当前的count不同时，触发重新渲染
       <button onClick={() => setCount(count + 1)}>
         Click me
       </button>
     </div>
   );
 }

    # 注意点

    • 在下一次重新渲染时，useState 返回给我们当前的 state。

    • setCount会异步地更新count的值，同时触发重新渲染。

    • 当setCount传入的参数和count相同时，不会触发重新渲染（浅比较）

    • 惰性初始state，传入一个函数，在函数中计算并返回初始的 state，此函数只在初始渲染时被调用。

       const [state, setState] = useState(() => {
   const initialState = someExpensiveComputation(props);
   return initialState;
 });

    • React 会确保更新函数 setState 的标识是稳定的，因此可以安全地从 useEffect 或 useCallback 的依赖列表中省略 setState

    # useEffect

    # 介绍

    useEffect可以在函数组件中执行副作用操作。通过使用这个 Hook，可以告诉 React 组件需要在渲染后执行某些操作。(使用了闭包机制)

     function Example() {
   const [count, setCount] = useState(0);
 
   useEffect(() => {
     document.title = `You clicked ${count} times`;
   });
 }

    • 模拟componentDidMount，第二参数为空数组。

 /** 模拟 componentDidMount 生命周期 */
 export const useMount = (fn: () => void) => {
   useEffect(() => {
     fn();
     // eslint-disable-next-line react-hooks/exhaustive-deps

    # 注意点

    • 每次重新渲染，都会执行上一个effect返回的清理函数，然后生成新的effect，替换掉之前的。
    • 确保第二参数的数组中包含了所有外部作用域中会随时间变化并且在 effect 中使用的变量。
    • 通常要在 effect 内部 去声明它所需要的函数。 这样就能容易的看出那个 effect 依赖了组件作用域中的哪些值。另一种方法是把函数加入 effect 的依赖但把它的定义包裹 进 useCallback Hook
    • React 提供了一个额外的 useLayoutEffect ，它和 useEffect 的结构相同，区别只是调用时机不同。它会在浏览器执行下一次绘制前被同步执行。
    • useEffect的回调参数返回的是一个清除副作用的 clean-up函数。因此无法返回 Promise，更无法使用 async/await，可以采用立即执行函数的方式封装一个useAsyncEffect解决该问题。
     // 定义
 export default function useAsyncEffect<T, U extends any[]>(
   method: () => Promise<T>,
   deps: U
 ) {
   useEffect(() => {
     (async () => {
       await method()
     })()
   }, deps)
 }
 
 // 使用
 export default function Demo() {
   const [count, setCount] = useState(0)
 
   function fetchData(): Promise<number> {
     return new Promise((resolve) => {
       setTimeout(() => {
         resolve(count + 1)
       }, 2000)
     })
   }
   useAsyncEffect(async () => {
     const count = await fetchData()
     setCount(count)
   }, [fetchData])
 
   return (
     <div>{count}</div>
   )
 }

    # useContext

    # 介绍

    Context 提供了一个无需为每层组件手动添加 props，就能在组件树间进行数据传递的方法，利用useContext可以做到局部数据的共享。

    接收一个 context 对象（React.createContext 的返回值）并返回该 context 的当前值。当前的 context 值由上层组件中距离当前组件最近的 <MyContext.Provider> 的 value prop 决定。

     const value = useContext(MyContext);
 const themes = {
   light: {
     foreground: "#000000",
     background: "#eeeeee"
   },
   dark: {
     foreground: "#ffffff",
     background: "#222222"
   }
 };
 
 const ThemeContext = React.createContext(themes.light);
 
 function App() {
   return (
     <ThemeContext.Provider value={themes.dark}>
       <Toolbar />
     </ThemeContext.Provider>
   );
 }
 
 function Toolbar(props) {
   return (
     <div>
       <ThemedButton />
     </div>
   );
 }
 
 function ThemedButton() {
   const theme = useContext(ThemeContext);  
   return (    
     <button style={{ background: theme.background, color: theme.foreground }}>      
       I am styled by theme context!    
     </button>  
   );

    # 注意点

    • 当组件上层最近的 <MyContext.Provider> 更新时，该 Hook 会触发重渲染，并使用最新传递给 MyContext provider 的 context value 值。即使祖先使用 React.memo 或 shouldComponentUpdate，也会在组件本身使用 useContext 时重新渲染。
    • 订阅了Context 对象的组件会从组件树中离自身最近的那个匹配的 Provider 中读取到当前的 context 值，只有当组件所处的树中没有匹配到 Provider 时，其 defaultValue 参数才会生效。
    • Context 主要应用场景在于很多不同层级的组件需要访问同样一些的数据，但它会使得组件的复用性变差。

    # React数据共享方案

    • 状态提升（嵌套层级深时，存在属性下钻的问题）
    • 路由传参

    • 组合组件（component composition）：将组件作为参数传递下去
    • 全局数据共享方案（Redux/Mobx/Hox）

    # useReducer

    # 介绍

     const [state, dispatch] = useReducer(reducer, initialArg, init);
    1

    useState 的替代方案。它接收一个形如 (state, action) => newState 的 reducer，并返回当前的 state 以及与其配套的 dispatch 方法。

    入参

    • reducer：形如 (state, action) => newState 的函数

    • initialArg：state的初始值

    • Init：惰性初始化state的函数，初始 state 将被设置为 init(initialArg)

     const initialState = {count: 0};
 
 function reducer(state, action) {
   switch (action.type) {
     case 'increment':
       return {count: state.count + 1};
     case 'decrement':
       return {count: state.count - 1};
     default:
       throw new Error();
   }
 }
 
 function Counter() {
   const [state, dispatch] = useReducer(reducer, initialState);
   return (
     <>
       Count: {state.count}
       <button onClick={() => dispatch({type: 'decrement'})}>-</button>
       <button onClick={() => dispatch({type: 'increment'})}>+</button>
     </>
   );
 }

    简化版的useReducer实现：

     function useReducer(reducer, initialState) {
   const [state, setState] = useState(initialState);
 
   function dispatch(action) {
     const nextState = reducer(state, action);
     setState(nextState);
   }
 
   return [state, dispatch];
 }

    # 注意点

    • React 会确保 dispatch 函数的标识是稳定的，因此可以安全地从 useEffect 或 useCallback 的依赖列表中省略 dispatch。

    # useMemo

    # 介绍

    把“创建”函数和依赖项数组作为参数传入 useMemo，它仅会在某个依赖项改变时才重新计算 memoized 值。

     const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
    1

    # 注意点

    • 传入 useMemo 的函数会在渲染期间执行，不要在函数内部执行副作用。
    • 如果没有提供依赖项数组，useMemo 在每次渲染时都会计算新的值。
    • useMemo可以作为性能优化的手段。

    # useCallback

    # 介绍

    传入一个函数和依赖项数组，它将返回该回调函数的 memoized 版本，该函数仅在某个依赖项改变时才会更新。useCallback(fn, deps) 相当于 useMemo(() => fn, deps)。

     const memoizedCallback = useCallback(
   () => {
     doSomething(a, b);
   },
   [a, b],
 );
    1
    2
    3
    4
    5

    和useMemo一样，useCallback常用作性能优化的手段。

    值得注意的是，在编写custom hook的时候，为避免页面的死循环渲染，常常需要使用useMemo和useCallback这两个hook。

    # useRef

    # 介绍

    useRef 返回一个可变的 ref 对象，其 .current 属性被初始化为传入的参数（initialValue）。返回的 ref 对象在组件的整个生命周期内持续存在。它类似于一个 class 的实例属性。

     function TextInputWithFocusButton() {
   const inputEl = useRef(null);
   const onButtonClick = () => {
     // `current` 指向已挂载到 DOM 上的文本输入元素
     inputEl.current.focus();
   };
   return (
     <>
       <input ref={inputEl} type="text" />
       <button onClick={onButtonClick}>Focus the input</button>
     </>
   );
 }

    useIsMounted：使用useRef和useEffect可以实现获取判断组件是否挂载的自定义hook

     /**
  * 用于判断组件是否已经挂载
  * @returns fn: 调用获取是否挂载布尔值
  */
 export const useIsMounted = () => {
   const ref = useRef(false);
   useEffect(() => {
     ref.current = true;
     return () => {
       ref.current = false;
     };
   }, []);
   return () => ref.current;
 };

    usePrevious：使用useRef和useEffect实现获取上一轮prop或state的自定义hook

  * 用于获取上一轮prop或state
  * @returns fn: 调用获取是否挂载布尔值

 function usePrevious(value) {
   const ref = useRef();
   // 渲染后执行
   useEffect(() => {
     ref.current = value;

   // 比useEffect内的函数先执行
   return ref.current;
 }

    # useImperativeHandle

    # 介绍

    useImperativeHandle 可以让你在使用 ref 时自定义暴露给父组件的实例值。在大多数情况下，应当避免使用 ref 这样的命令式代码。useImperativeHandle 大多数情况下会与用于转发Ref的forwardRef 一起使用。

     function FancyInput(props, ref) {
   const inputRef = useRef();
   useImperativeHandle(ref, () => ({
     focus: () => {
       inputRef.current.focus();
     }
   }));
   return <input ref={inputRef} ... />;
 }
 FancyInput = forwardRef(FancyInput);

    # useLayoutEffect

    # 介绍

    其函数签名与 useEffect 相同，但它会在所有的 DOM 变更之后同步调用 effect。可以使用它来读取 DOM 布局并同步触发重渲染。尽可能使用标准的 useEffect 以避免阻塞视觉更新。

    # React.memo优化

    用 React.memo 包裹一个组件来对它的 props 进行浅比较

     const Button = React.memo((props) => {
   // 你的组件
 });

    3

    React.memo 等效于 PureComponent，但它只比较 props。（可以通过第二个参数指定一个自定义的比较函数来比较新旧 props。如果函数返回 true，就会跳过更新。）

    # useMemo优化

    useMemo 允许你跳过一次子节点的昂贵的重新渲染：

     function Parent({ a, b }) {
   // Only re-rendered if `a` changes:
   const child1 = useMemo(() => <Child1 a={a} />, [a]);
   // Only re-rendered if `b` changes:
   const child2 = useMemo(() => <Child2 b={b} />, [b]);
   return (
     <>
       {child1}
       {child2}
     </>
   )
 }
